# AI工作台三端协同修改方案（领导交流意见修订版）

> 基于 2026-05-26 领导交流意见及后续细化要求制定

---

## 一、整体原则

1. **底座一体化**：三端共用公司自动驾驶大模型底座，不是三个独立系统。
2. **AI交互式**：以自然语言对话为核心，助手即提示词，拒绝传统 Table+按钮+表单的 OA 形态。
3. **三类角色视图**：员工端（展业人员）、业务支持端（支持中心人员）、协同端（组织者/决策层），一人可多角色。
4. **沿用已评审设计**：员工端直接复用已评审通过的原有设计，不在其上做大的风格改动。

## 三、业务支持端修改方案

### 3.1 设计思路

与员工端共用同一套 AI 交互框架：顶部助手矩阵 → 中间 AI 对话区域 → 底部输入框。区别在于：
- 顶部助手名字为「XX支持中心」
- 对话区域只展示 AI 推送的「今日任务」，不展示具体事项详情
- 整体风格与员工端保持一致（沿用已评审版的配色和布局逻辑）

### 3.2 具体修改点

| 序号 | 修改项 | 具体做法 |
|:---:|---|---|
| 1 | **顶部 7 个助手** | 与员工端顶部 7 个智能体对应，但名字改为：<br>① 投行支持中心<br>② 资管支持中心<br>③ 零售支持中心<br>④ 投资支持中心<br>⑤ 销交支持中心<br>⑥ 机构支持中心<br>⑦ 交叉验证中心 |
| 2 | **删除传统侧边栏+Table** | 删除现有 SupportPage 的左侧业务条线侧边栏、中间 Table 列表、状态标签、操作按钮。 |
| 3 | **AI 对话区域（上）** | 7 个助手下方是 AI 对话区域。对话区域顶部展示 AI 推送的「今日任务」卡片（如"今日有 3 项待复核事项"），不展开具体事项内容。 |
| 4 | **AI 输入框（下）** | 对话区域下方是全局输入框，与员工端保持一致。输入框 placeholder 改为「输入问题，或点击上方支持中心快捷提问」。 |
| 5 | **助手点选逻辑** | 点击顶部某个支持中心头像，等同于在输入框填入该支持中心的提示词前缀，直接进入对话态。 |
| 6 | **共享协同** | 支持端不用龙虾（个人空间），信息落在统一语料库，做完即同步。 |

### 3.3 业务支持端页面结构（最终）

```
│  [业务支持端 Logo]  员工端 | 业务支持端 | 协同端  │  ← 三端切换 Tab
├─────────────────────────────────────────────┤
│  [投行支持中心]...[交叉验证中心]              │  ← 顶部 7 个支持中心
├─────────────────────────────────────────────┤
│  🤖 早安，今日有 3 项待复核任务              │  ← AI 对话区域（今日任务推送）
│  ├─ 任务 1：贵州茅台定增方案待复核           │
│  └─ 任务 2：合同录入异常检测（3份）          │
├─────────────────────────────────────────────┤
│  [输入框] 输入问题，或点击上方支持中心快捷提问  │  ← 底部输入框
└─────────────────────────────────────────────┘
```

## 四、协同端修改方案

- 顶部几个协同类智能体入口
- 下方是 AI 对话区域 + 输入框
- 对话区域中预置 5 个维度的提示词卡片，用户可点击直接发起对话

### 4.2 具体修改点

| 序号 | 修改项 | 具体做法 |
|:---:|---|---|
| 1 | **删除五大板块** | 删除 CollabPage 原有的「请示事项」「经营看板」「风险提示」「队伍状况」「应急组织」五个独立 Section。 |

## 七、关键注意事项

1. **员工端尽量不动原有设计**：领导明确要求"尽量不要去动原来那个基本的设计"，整合已评审版本后，只在其上做最小必要调整（5 个助手、去掉登录按钮、改为今日任务）。
2. **7 个顶部智能体要对应**：员工端、业务支持端、协同端的顶部智能体要有对应关系，体现"三类角色视图"的概念。
3. **5 个助手顺序严格固定**：员工端的 5 个助手顺序（客户分析→业务分析→方案生成→交叉验证→客户服务）不可更改。
4. **提示词即交互**：三个端的核心交互都是提示词驱动的，不要回到传统按钮+表单的模式。
5. **协同端不要堆砌看板**：领导明确说"请示事项不是重点，哪有那么多请示"，通过 AI 对话+提示词卡片的方式自然呈现即可。
