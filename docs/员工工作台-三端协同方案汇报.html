三端定位说明
端一 · 员工端

员工端

使用角色：各条线员工
以工作提示为核心，聚合员工日常所需的预警、业绩、AI智能体等功能，让员工在一个页面内完成工作感知与AI辅助作业。
  • 工作提示驱动注意力管理
  • 四大AI智能体一键唤起
  • 个人业绩数据实时呈现
  • 提交事项至支持端处理
端二 · 新建

业务支持端

使用角色：各条线支持中心人员
不管人，只管业务动作。负责接收员工端提交的事项，经AI路由与质控后处置，需跨部门决策则提请协同端。
  • 按条线分类管理事项队列
  • AI自动预判优先级与分类
  • 质控/复核/规则知会闭环
  • 将高优先级事项推送协同端
端三 · 新建

协同端

使用角色：分管领导 / 决策层
汇聚需要跨部门协同和领导决策的事项与议题，支持批准执行和批注意见，形成有记录、可追溯的决策闭环。
  • 请示事项集中批阅
  • 业务协同与风险议题决策
  • 应急组织快速响应
  • AI会议纪要与决策摘要
三大业务闭环
闭环一：员工独立完成 最常见
员工发起业务 → AI智能体处理 → 员工确认/输出 → 完成
支持端在后台自动感知（规则触达），不介入处理
闭环二：员工→支持中心协同 需质控 / 复核
员工主动提交工单 → AI预判+分类路由 → 支持中心看板接单 → 决策/质控/复核 → 完成
支持端也可主动前置介入（规则触发），无需等员工提交
闭环三：支持中心→圆桌会议协同 跨部门 / 资源申请
支持端/员工提请协同 → 规则研判+议题生成 → 圆桌会议议题 → 高级领导决策 →
指令下发至相关端
三个闭环可并行触发、不强制串联，员工端也可直接提请协同端（越级申请）

设计要点

零学习成本：工作提示按紧急/风险/进展/待办/日程五类聚合，员工打开页面即可感知当日重点工作，无需培训。

AI 无缝嵌入：四大智能体以卡片形式常驻页面，点击直接对话；全局输入框支持自然语言唤起任意智能体，减少操作路径。

一键提交事项：工作提示可直接转化为事项提交至业务支持端，形成"感知→处理→反馈"闭环。

业务支持端
不管人，只管业务动作。负责接收员工端提交的事项，经AI路由与质控后处置，需跨部门决策则提请协同端。
核心定位
事项的路由中心与处置中心；AI预判分类，质控复核，规则知会，处置结果回传。
主要功能模块
事项列表 · AI路由预判 · 质控复核 · 追踪看板 · 规则知会下发
覆盖条线
资产管理条线 · 投资银行条线 · 零售财富条线 · 合规风控条线
设计要点

AI 预判 + 自动路由：事项进入支持端后，AI 自动识别类型与优先级并推荐路由目标，减少人工分拣成本，支持一键确认或手动调整。

质控复核闭环：处置过程全程留痕，质控结果、规则知会记录与处置结论绑定同一事项，确保可追溯、可审计。

跨部门协同入口：遇需决策或跨部门协调的事项，支持端可直接提请至协同端，形成"接收→处置→提请→反馈"完整链路。

协同端
根据当前登录人角色动态展示待处理事项与关注数据。业务负责人和分管领导可在此处理请示事项、批注意见、下发指令，也可查看规则触发的知晓类事项及执行进展。
核心定位
面向业务负责人与决策层的动态工作台；按角色权限聚合待决策事项与需关注数据，支撑批阅、指令、跟踪全链路。
主要功能
待决策事项 · 批注意见 · 指令下发 · 知晓事项 · 执行进展跟踪
交互方式
点击事项查看全面信息 → 批注意见/批准执行/下发指令 → 跟踪后续进展
设计要点

角色动态视图：根据登录人身份自动筛选其权责范围内的事项，不同领导看到的待办与关注数据各不相同，避免信息过载。

决策操作闭环：请示事项支持点击查看完整信息后，进行批注意见、批准执行或下发指令，决策记录自动归档并同步相关端。

主动知晓机制：除待决策事项外，系统按规则触发需领导知晓关注的事项推送，领导可随时查看执行进展，无需被动等待汇报。
