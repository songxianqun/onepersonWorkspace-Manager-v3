员工工作台三端协同 - 设计方案

基于518智能体项目会议纪要 · 公司级员工工作台三端建设
三端不是三个系统，而是同一平台的三类角色视图
员工工作台基于518智能体项目会议纪要，在公司级层面建设三端协同体系。通过统一的事项数据模型贯通三端，同一应用内通过顶部 Tab 切换视图。员工端聚焦个人工作提示与AI智能体调用；业务支持端负责事项的路由分发与质控处理；协同端汇聚跨部门议题，由决策层进行批准与批注。三端形成完整闭环，覆盖"提交—处理—决策"全链路。

三端定位说明
端一 · 员工端

员工端

使用角色：各条线员工
以工作提示为核心，聚合员工日常所需的预警、业绩、AI智能体等功能，让员工在一个页面内完成工作感知与AI辅助作业。
  • 工作提示驱动注意力管理
  • 四大AI智能体一键唤起
  • 个人业绩数据实时呈现
  • 提交事项至支持端处理
端二 · 新建

业务支持端

使用角色：各条线支持中心人员
不管人，只管业务动作。负责接收员工端提交的事项，经AI路由与质控后处置，需跨部门决策则提请协同端。
  • 按条线分类管理事项队列
  • AI自动预判优先级与分类
  • 质控/复核/规则知会闭环
  • 将高优先级事项推送协同端
端三 · 新建

协同端

使用角色：分管领导 / 决策层
汇聚需要跨部门协同和领导决策的事项与议题，支持批准执行和批注意见，形成有记录、可追溯的决策闭环。
  • 请示事项集中批阅
  • 业务协同与风险议题决策
  • 应急组织快速响应
  • AI会议纪要与决策摘要
三大业务闭环
1
事项提交与处置闭环
员工通过工作提示感知事项，提交至业务支持端，经AI路由质控后处置完成，结果反馈至员工端。
员工感知 → 提交事项 → AI路由质控 → 处置完成 → 结果反馈
2
跨部门协同与决策闭环
支持端发现跨部门事项，提请协同端，由决策层批准执行或批注意见，指令下发后支持端执行落地。
支持端识别 → 提请协同 → 领导批阅 → 下发执行 → 闭环归档
3
应急响应与处置闭环
协同端应急组织模块集中呈现突发事件，领导可快速查看时间线与影响，批准启动应急预案，全程留痕。
应急事件触发 → 协同端汇聚 → AI摘要 → 批准预案 → 执行归档

事项核心字段说明
字段 类型 说明 涉及端
id string 事项唯一标识 三端共享
type enum 事项类型：urgent / risk / progress / todo / schedule 员工端触发
priority enum 优先级：P1紧急 / P2重要 / P3普通 支持端AI判定
status enum 处置状态：pending / routing / processing / reviewing / decided / closed 三端流转
routeTo string AI路由至的支持中心名称 支持端
decision object 决策记录：{ action, actor, time, comment } 协同端
timeline array 操作时间线：[{ time, actor, action, note }] 三端可查
aiSummary string AI 生成的事项摘要，用于协同端快速阅读 协同端
议题数据模型（协同端专属）
字段 类型 说明
agendaId string 议题唯一标识
section enum 所属板块：agenda / business / risk / team / emergency
source string 来源端与提请人
tags array 标签：[优先级标签, 类型标签]，最多2个
decisionActions enum[] 可用决策动作：approve（批准执行）/ comment（批注意见）
emergencyLevel enum 应急等级：P1 / P2 / P3（仅应急组织模块）
timeline array 事件时间线，可展开查看（可折叠）
